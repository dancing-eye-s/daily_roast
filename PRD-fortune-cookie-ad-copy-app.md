# PRD: 하루 한 번 광고 카피 포춘쿠키 앱

## 1. 제품 개요

사용자가 하루에 한 번 포춘쿠키를 누르면 대한민국광고대상 수상작에서 선별한 광고 카피 문구 1개를 랜덤으로 보여주는 웹/모바일 앱이다. 사용자는 매일 짧고 인상적인 카피를 통해 영감, 위트, 관점 전환을 얻고, 광고 카피의 출처와 맥락을 함께 확인할 수 있다.

## 2. 목표

- 하루 1회만 열 수 있는 간단한 의식형 경험을 만든다.
- 대한민국광고대상 수상작 기반의 신뢰도 높은 카피 큐레이션을 제공한다.
- 광고 카피를 단순 소비가 아니라 학습과 영감의 소재로 경험하게 한다.
- 향후 저장, 공유, 주제별 큐레이션, 카피 분석 기능으로 확장 가능한 구조를 만든다.

## 3. 배경 및 근거

대한민국광고대상은 한국광고총연합회가 주최/주관하는 국내 주요 광고상으로, 연도별 수상작 발표 자료와 수상작 리스트가 광고정보센터 및 관련 공식/언론 자료를 통해 일부 공개되어 있다. PRD 기준 데이터 후보 출처는 다음과 같다.

- 한국광고총연합회/광고정보센터의 대한민국광고대상 수상작 발표 자료
- 연도별 수상작 PDF 또는 첨부 리스트
- 수상 광고주의 공식 보도자료, 캠페인 페이지, 영상 설명
- 신뢰 가능한 언론 기사

중요: 광고 카피 문구는 창작물이며 브랜드/광고회사/제작사의 권리가 존재할 수 있다. 따라서 실제 서비스에는 권리 확인, 라이선스 확보, 짧은 인용 범위 검토, 출처 표기, 삭제 요청 프로세스가 필요하다.

## 4. 타깃 사용자

- 광고/마케팅/브랜딩 실무자
- 카피라이터, 기획자, 디자이너, 콘텐츠 제작자
- 좋은 문장과 브랜드 메시지에서 영감을 얻고 싶은 일반 사용자
- 매일 짧은 루틴으로 창의적 자극을 얻고 싶은 사용자

## 5. 핵심 사용자 시나리오

1. 사용자가 앱에 접속한다.
2. 오늘 아직 포춘쿠키를 열지 않았다면 닫힌 포춘쿠키가 표시된다.
3. 사용자가 포춘쿠키를 누른다.
4. 쿠키가 열리는 애니메이션과 함께 광고 카피 1개가 랜덤으로 표시된다.
5. 사용자는 카피, 브랜드/캠페인명, 수상 연도, 수상 부문, 출처를 확인한다.
6. 오늘의 카피는 하루 동안 다시 볼 수 있다.
7. 다음 포춘쿠키까지 남은 시간이 표시된다.

## 6. MVP 범위

### 포함

- 하루 1회 포춘쿠키 열기
- 랜덤 카피 1개 노출
- 오늘의 카피 재방문 시 동일 문구 유지
- 다음 오픈 가능 시간 표시
- 카피 상세 정보 표시
- 기본 공유 기능
- 로컬 스토리지 기반 일일 제한
- 초기 수동 큐레이션 데이터셋

### 제외

- 로그인/회원가입
- 서버 기반 사용자 이력 저장
- 광고 카피 자동 크롤링
- AI 카피 생성
- 카피 즐겨찾기/컬렉션
- 푸시 알림
- 관리자 CMS

## 7. 기능 요구사항

### 7.1 포춘쿠키 열기

- 사용자는 하루에 한 번만 포춘쿠키를 열 수 있다.
- 하루 기준은 기본적으로 사용자의 로컬 타임존 날짜로 계산한다.
- 포춘쿠키를 열면 당일 선택된 카피 ID를 저장한다.
- 같은 날 재접속하면 새 카피를 뽑지 않고 당일 카피를 다시 보여준다.
- 다음 날 00:00 이후 새 포춘쿠키를 열 수 있다.

### 7.2 랜덤 선택

- 전체 활성 카피 목록에서 1개를 무작위 선택한다.
- 최근 N일 내 본 카피는 가능하면 제외한다.
- MVP에서는 로컬에 최근 본 카피 ID 목록을 저장한다.
- 카피 수가 적어 제외 조건을 만족할 수 없으면 전체 목록에서 선택한다.

### 7.3 카피 카드

카피 결과 화면은 다음 정보를 표시한다.

- 광고 카피 문구
- 브랜드명
- 캠페인/작품명
- 대한민국광고대상 수상 연도
- 수상 부문 및 상격
- 광고회사/제작사, 확보 가능한 경우
- 출처 링크
- 짧은 맥락 설명

### 7.4 공유

- 사용자는 오늘의 카피를 텍스트 또는 이미지 카드 형태로 공유할 수 있다.
- MVP에서는 Web Share API를 우선 사용한다.
- 공유 문구에는 앱 이름과 출처성 메타데이터를 포함한다.
- 저작권 검토 전에는 전체 카피 공유 대신 앱 링크 공유를 기본값으로 둔다.

### 7.5 잠금 상태

- 이미 오늘 포춘쿠키를 열었다면 결과 화면을 보여준다.
- 새로 뽑기는 비활성화한다.
- 다음 포춘쿠키까지 남은 시간을 표시한다.

## 8. 콘텐츠 요구사항

### 8.1 데이터 필드

```json
{
  "id": "string",
  "copy": "string",
  "brand": "string",
  "campaignTitle": "string",
  "awardName": "대한민국광고대상",
  "awardYear": 2024,
  "category": "string",
  "prize": "대상 | 금상 | 은상 | 동상 | 특별상",
  "advertiser": "string",
  "agency": "string",
  "productionCompany": "string",
  "sourceTitle": "string",
  "sourceUrl": "string",
  "rightsStatus": "verified | pending | excerpt_only | unavailable",
  "note": "string",
  "tags": ["string"],
  "isActive": true
}
```

### 8.2 콘텐츠 수집 원칙

- 수상작 메타데이터는 공식 자료 또는 신뢰 가능한 기사에서 확인한다.
- 카피 문구는 공식 이미지, 영상, 보도자료, 광고주 공식 채널에서 확인한다.
- 출처 URL과 확인일을 기록한다.
- 권리 상태가 불명확한 카피는 서비스 노출 전 `pending`으로 둔다.
- 문제 제기 시 즉시 비활성화할 수 있어야 한다.

### 8.3 초기 데이터셋

- MVP 목표: 50개 이상
- 베타 목표: 150개 이상
- 정식 출시 목표: 365개 이상

하루 1회 앱이므로 365개 이상 확보하면 1년 동안 매일 다른 카피를 제공할 수 있다.

## 9. UX 요구사항

- 첫 화면은 닫힌 포춘쿠키가 중심이 되는 단순한 구성으로 한다.
- 사용자는 설명을 읽지 않아도 무엇을 눌러야 하는지 알 수 있어야 한다.
- 쿠키를 누르는 순간 짧은 기대감이 생기도록 0.8~1.5초 애니메이션을 제공한다.
- 결과 화면은 카피 문구가 가장 크게 보이도록 한다.
- 출처와 수상 정보는 보조 정보로 간결하게 표시한다.
- 잠금 상태는 답답하지 않게 “오늘의 문구를 다시 볼 수 있는 공간”처럼 표현한다.

## 10. 화면 구성

### 10.1 홈/열기 전

- 앱 제목
- 닫힌 포춘쿠키 비주얼
- 열기 버튼 또는 쿠키 직접 탭 인터랙션
- 짧은 보조 문구

### 10.2 결과 화면

- 열린 포춘쿠키 비주얼
- 오늘의 광고 카피
- 브랜드/캠페인/수상 정보
- 공유 버튼
- 출처 보기 버튼
- 다음 오픈까지 남은 시간

### 10.3 출처/상세 모달

- 작품명
- 광고주/브랜드
- 광고회사/제작사
- 수상 연도/부문/상격
- 출처 링크
- 권리/출처 안내 문구

## 11. 비기능 요구사항

- 모바일 우선 반응형 웹
- 초기 로딩 2초 이하
- 오프라인 상태에서도 이미 받은 오늘의 카피는 표시
- 로컬 스토리지 장애 시 기본 동작 유지
- 접근성: 키보드 조작, 충분한 색 대비, 스크린리더 라벨 제공
- 개인정보 최소 수집: MVP에서는 계정/개인정보 수집 없음

## 12. 분석 지표

- 일일 방문자 수
- 포춘쿠키 오픈율
- 재방문율 D1/D7/D30
- 공유 버튼 클릭률
- 출처 보기 클릭률
- 카피별 반응률
- 하루 제한 상태에서의 재방문 수

## 13. 정책 및 법적 고려사항

- 광고 카피 원문 사용 전 저작권/상표권/초상권/광고주 권리 검토가 필요하다.
- 출처를 명확히 표시한다.
- 원문 전체가 아닌 짧은 인용 또는 요약형 큐레이션을 기본 정책으로 검토한다.
- 브랜드가 삭제를 요청할 경우 즉시 비활성화한다.
- 상업 서비스화 전 법률 검토를 진행한다.

## 14. 성공 기준

- 사용자가 5초 안에 앱의 핵심 행동을 이해한다.
- 첫 방문자의 70% 이상이 포춘쿠키를 연다.
- 베타 사용자 D7 재방문율 20% 이상을 달성한다.
- 초기 데이터셋 50개 이상이 출처와 함께 검수된다.
- 권리 미확인 콘텐츠가 실서비스에 노출되지 않는다.

## 15. 향후 확장

- 로그인 기반 히스토리/즐겨찾기
- “오늘의 카피 해석” 기능
- 업종별/감정별/카피 유형별 필터
- 월간 카피 캘린더
- 사용자 투표: 오늘의 카피가 준 느낌
- 광고대상 수상작 아카이브
- 관리자용 콘텐츠 CMS
- 푸시 알림 또는 이메일 리마인더
